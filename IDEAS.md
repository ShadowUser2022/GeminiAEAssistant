# 💡 Gemini AE Assistant — Feature Backlog & Ideas

Этот файл служит копилкой наших технологических идей и улучшений для панели Gemini AE Assistant. Любой новый чат или ассистент может брать задачи отсюда для реализации.

---

## 🚀 Активные идеи для реализации

### 1. Голосовой ввод промптов (Web Speech API Dictation Button)
*   **Проблема:** Встроенная диктовка macOS блокируется After Effects и контейнером CEP из-за перехвата фокуса ввода клавиатуры.
*   **Решение:** Добавить маленькую неоновую кнопку-микрофон прямо в поле ввода чата.
*   **Реализация:**
    *   При клике запускать встроенное в Chromium браузерное распознавание `webkitSpeechRecognition`.
    *   Отображать пульсирующий индикатор записи в поле ввода.
    *   По окончании речи автоматически вставлять распознанный текст в `promptInput`.
    *   *Плюсы:* Работает на 100% кроссплатформенно, не зависит от фокуса macOS/Windows и не требует внешних библиотек.

### 2. Визуальное превью прикрепленного контекста
*   **Идея:** При наведении на чипс-ссылку выделенного слоя (в нашем новом `context-chips-container`) показывать маленькую всплывающую карточку с дополнительной информацией о слое:
    *   Тип слоя (Text, Shape, Solid, Footage).
    *   Время старта/конца на таймлайне.
    *   Активные эффекты, примененные к слою.
*   **Реализация:** Расширить ExtendScript функцию `getSelectedContext()`, чтобы она собирала эти метаданные, и выводить их в HTML-атрибуты чипса.

### 3. Интеллектуальный поиск по истории (Smart Log Viewer)
*   **Идея:** Добавить вкладку или выдвижную панель со списком прошлых сгенерированных скриптов.
*   **Реализация:** Читать файлы логов из папки `/logs` и выводить красивый интерактивный список с кнопкой «Применить повторно».

### 4. Динамическое скользящее окно контекста (Smart Context Sliding Window)
*   **Проблема:** В `js/config.js` жестко задан лимит `CONTEXT_MAX_CHARS = 16000` (~4,000 токенов). В современных моделях (Gemini Pro, Claude 3.5 Sonnet) размер контекста составляет от 200K до 2M токенов, поэтому индикатор в панели почти мгновенно становится красным, вводя пользователя в заблуждение, хотя фактического переполнения нет. При этом неограниченный рост истории чата приводит к замедлению ответов и лишним расходам API.
*   **Решение:** Разработать адаптивный менеджер контекста, который динамически регулирует объем отправляемой истории в зависимости от выбранной модели и ее тарифа (Free/Paid).
*   **Реализация:**
    *   **Лимиты по моделям:**
        *   *Бесплатные модели* (например, Gemini Flash Lite) — лимит ~60,000 символов для предотвращения ошибок лимита запросов в минуту (TPM - Tokens Per Minute).
        *   *Платные модели* (Claude 3.5 Sonnet, Gemini 1.5 Pro) — расширенный лимит до 300,000–500,000 символов для глубокого понимания контекста проекта.
    *   **Алгоритм обрезки (Sliding Window):**
        *   Формировать payload запроса с конца истории (от самых свежих сообщений к более старым).
        *   Суммировать длину текстовых сообщений и прикрепленных файлов изображений (с учетом их веса `IMAGE_CONTEXT_WEIGHT`).
        *   Как только суммарный объем превышает лимит выбранной модели, отсекать более старые сообщения из отправляемого пакета.
    *   **Обновление UI:**
        *   Динамически пересчитывать заполнение шкалы контекста (`updateContextTracker`) при переключении моделей в `#modelSelect`.
        *   Показывать реальный лимит в процентах на круговой шкале.
    *   *Плюсы:* Максимальная скорость работы API, защита от перерасхода токенов на платных тарифах, отсутствие ошибок TPM на бесплатных тарифах и всегда актуальный контекст без засорения старой историей.
